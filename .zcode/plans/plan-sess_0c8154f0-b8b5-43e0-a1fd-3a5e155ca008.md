### 3b. Cross-platform improvements

- Add FreeBSD target using cross-compilation from Linux
- For BSD Family: document that Linux x86_64 binaries work on FreeBSD via Linux ABI compatibility layer, alongside native FreeBSD builds
- Improve artifact upload to include all formats (dmg, app, deb, rpm, AppImage, msi, exe, and add .tar.gz for FreeBSD)

### 3c. Harmony OS strategy

**Note:** Tauri v2 does **not** officially support Harmony OS. There are two practical paths:

| Approach          | What                                            | Pros                                                        | Cons                                                               |
| ----------------- | ----------------------------------------------- | ----------------------------------------------------------- | ------------------------------------------------------------------ |
| **Web / PWA**     | Deploy the Vite frontend as a PWA               | Works on any browser including Harmony OS; no extra tooling | Limited native fs access; needs separate backend                   |
| **Android build** | Target `aarch64-linux-android` via Tauri mobile | Can run on Harmony OS devices that support Android apps     | Harmony OS NEXT dropped Android compatibility; complex build setup |

**Recommendation:** Phase 1 — add the PWA approach so Harmony OS users can access the app via browser. Phase 2 — once Tauri adds official Harmony OS support (or OHOS Tauri community matures), add native builds. Document this roadmap in README.
