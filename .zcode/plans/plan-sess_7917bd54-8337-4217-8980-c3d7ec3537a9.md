## Plan: Dual-Language Settings - Fixing the Gaps

### Current State

The architecture for **two languages** (App UI language + AI Output language) already exists in the database schema:

- `user_profile.preferred_ui_language` ✅
- `user_profile.preferred_intelligence_language` ✅
- `PATCH /profile` API accepts both ✅
- `ProfileService.generateSummary()` and `improveInstruction()` correctly use `preferredIntelligenceLanguage` ✅

### 4 Gaps Found

---

### Gap 1 🔴: `SearchService.ask()` hardcodes `outputLanguage: "en"`

**File:** `apps/core-engine/src/modules/search/search.service.ts` (line 231)

The RAG search always answers in English regardless of profile setting. The service already reads `customInstruction` from the profile synchronously — add a `readIntelligenceLanguage()` method using the same pattern.

**Change:**

```typescript
// Add method (line ~270):
private readIntelligenceLanguage(): string {
  const row = this.db.db
    .select({ lang: userProfile.preferredIntelligenceLanguage })
    .from(userProfile)
    .where(eq(userProfile.id, "default"))
    .get();
  return (row?.lang ?? "en").trim() || "en";
}

// Change line 231 from:
outputLanguage: "en",
// To:
outputLanguage: this.readIntelligenceLanguage(),
```

### Gap 2 🔴: `IntelligenceService.generate()` and `summarizePeriod()` default to `"en"`

**File:** `apps/core-engine/src/modules/intelligence/intelligence.service.ts`

Both methods default `targetLanguage` to `"en"` when not provided. Several callers (AnalyzingPage, BriefPage, PeriodSummaryPanel) don't pass it. Fix: when `targetLanguage` is omitted, read from profile.

**Change:**

- Import `userProfile` from schema
- Add `readIntelligenceLanguage()` private method
- In `generate()`: `const targetLanguage = opts.targetLanguage ?? this.readIntelligenceLanguage();`
- In `summarizePeriod()`: `const targetLanguage = opts.targetLanguage ?? this.readIntelligenceLanguage();`

---

### Gap 3 🟡: No UI control for `preferredIntelligenceLanguage` on Profile page

**New component:** `AiLanguageSection` — added to `ProfilePage.tsx`

A new section with a dropdown selector of common BCP-47 codes:
en, fa, es, fr, de, ja, zh-CN, ar, ru, pt, tr, it, ko, hi

- Reads current value from profile query
- On change, calls `patchProfile({ preferredIntelligenceLanguage: code })`
- Positined after `LanguageSection` on the Profile page

**English catalog** (`en.ts`): adds translation keys under `profile.aiLanguage`, `profile.aiLanguageHint`

### Gap 4 🟡: `LanguageSection` doesn't sync `preferredUiLanguage` to backend

**File:** `apps/desktop/src/components/shell/LanguageSection.tsx`

When user selects a different UI language, only the frontend i18n store updates. Need to also save to backend.

**Change:**

- Accept new optional prop `onLocaleChange?: (code: string) => void`
- Call `onLocaleChange(code)` inside the existing `setActive(code)` flow
- In `ProfilePage.tsx`, pass `onLocaleChange={(code) => patchProfile({ preferredUiLanguage: code })}`
