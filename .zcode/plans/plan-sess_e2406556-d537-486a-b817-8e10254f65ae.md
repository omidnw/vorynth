# Changelog Page тАФ Implementation Plan

## Overview

A new `/changelog` route (inside the shell) that lists version releases with themed codenames drawn from the Vorynth brand language. Each release is a `GhostCard` section with version number, codename, date, and categorized change entries (New / Improved / Fixed / Security) using the existing `ImportanceBadge` / `DomainTag` primitives for visual consistency.

## Files to create

### 1. `apps/desktop/src/features/changelog/changelog-data.ts`

A typed data file holding the release entries тАФ easy to update by appending a new object. Structure:

```ts
interface ChangeEntry {
	type: "new" | "improved" | "fixed" | "security";
	text: string;
}
interface Release {
	version: string; // "1.0.0"
	codename: string; // "Signal Over Noise" (brand-themed)
	date: string; // "2026-07-19"
	summary: string; // one-line headline
	changes: ChangeEntry[];
}
```

**v1.0.0 тАФ "Signal Over Noise"** will cover everything built so far: news-first design, multi-source collection, LangGraph intelligence workflow, 4 LLM providers, rate limiting, background jobs, search (keyword + AI RAG), citations, backup/restore, i18n + RTL, dark mode, Tauri shell.

### 2. `apps/desktop/src/pages/ChangelogPage.tsx`

Modeled on `DesignSystemPage.tsx`:

- **Header:** eyebrow ("Release Notes"), headline (`Changelog`), subtitle (tagline)
- **Current version badge:** reads `EngineStatus.version` from the existing `["engine-status"]` React Query (same hook SettingsPage uses), shown as a badge next to the latest release
- **Body:** `space-y-16` of `GhostCard` sections, one per release. Each card contains:
  - Version number + codename (`font-headline text-headline-md text-primary`)
  - Date + summary (`font-body text-body-md text-on-surface-variant`)
  - A list of changes, each prefixed with a type badge:
    - `new` тЖТ `ImportanceBadge tier="signal"` "New"
    - `improved` тЖТ `ImportanceBadge tier="trend"` "Improved"
    - `fixed` тЖТ `DomainTag` "Fixed"
    - `security` тЖТ `DomainTag` "Security"
  - The change text in `font-body text-body-md text-on-surface`

## Design decisions

- **Static data, not API-driven** тАФ the changelog describes the app itself, not user data. A TS file is simpler and ships with the bundle. Adding a new release = appending an object.
- **Brand-themed codenames** тАФ version names draw from project-details.md philosophy: "Signal Over Noise", "Precision Minimalism", "Obsidian Intelligence", "Explain Don't Just Summarize", etc.
- **Current version from engine status** тАФ reuses the existing query, no new endpoint needed.
- **Icon `change_history`** тАФ a Material Symbols icon that visually represents a changelog/history; fits the Precision Minimalism aesthetic.
