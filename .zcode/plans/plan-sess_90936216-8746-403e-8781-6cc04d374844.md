# Plan: Fix Windows ESM crash + Solve Node.js dependency

## Issue 1: Windows `ERR_UNSUPPORTED_ESM_URL_SCHEME` (the error you shared)

**Root cause** — `apps/core-engine/scripts/bundle-sidecar.mjs` generates `launcher.cjs` which does:
```javascript
const bundle = path.join(__dirname, "index.js");
import(bundle).catch(...);
```
On Windows, `path.join(__dirname, "index.js")` produces `c:\Users\...\index.js`. The ESM `import()` interprets `c:` as a URL scheme and rejects it — only `file:`, `data:`, and `node:` are valid.

**Fix** — Use `pathToFileURL()` from Node's `url` module:
```javascript
const { pathToFileURL } = require('url');
const bundle = pathToFileURL(path.join(__dirname, "index.js")).href;
```
This converts `c:\path\to\index.js` → `file:///c:/path/to/index.js` which the ESM loader accepts.

**Files to modify:**
1. `apps/core-engine/scripts/bundle-sidecar.mjs` — update the launcher template
2. `apps/desktop/src-tauri/binaries/vorynth-core-aarch64-apple-darwin/launcher.cjs` — apply the same fix to the already-built local bundle

## Issue 2: App requires Node.js installed (your bigger concern)

Your CI workflow comment at package.yml:18 already says:
> "for now the installers bundle Node via Tauri's resource mechanism"

But no actual Node.js binary is bundled — only the ncc JS bundle is shipped. Two approaches to solve this:

### Approach A: Bundle portable Node.js (simpler)
- Download the official portable/standalone Node.js binary for each target platform in CI
- Place it inside the sidecar directory alongside `launcher.cjs`
- Update Rust `sidecar_command()` → try bundled `node` first, then search PATH
- **Size increase**: ~40–50 MB per platform
- **CI changes**: Add a download + copy step before "Stage sidecar for Tauri"
- **Rust changes**: Small update to `which_node()` or `sidecar_command()`

### Approach B: Node.js SEA (Single Executable Application, already planned)
- Use Node.js 20+'s built-in SEA feature (your CI already has Node 24)
- After ncc bundling, create a standalone `vorynth-core` / `vorynth-core.exe` binary
- Works exactly like the Rust code's Strategy 1 (already implemented, just needs the binary)
- Still needs `better_sqlite3.node` alongside as a separate file (native addons can't be inlined)
- **Size increase**: ~45–50 MB per platform (same ballpark as bundled Node)
- **CI changes**: Add SEA build step (config + inject) after the bundle step
- **Rust changes**: None — Strategy 1 already looks for `vorynth-core[-.exe]`

### My recommendation
**Approach A (bundle portable Node.js)** is simpler to implement and more reliable. Approach B (SEA) is cleaner but has more edge cases with native addons. Both are similar in size impact since SEA includes the full Node runtime anyway.
