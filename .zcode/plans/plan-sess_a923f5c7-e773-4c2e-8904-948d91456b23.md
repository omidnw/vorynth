## Full-Text Search با رویکرد Hybrid + کتابخانه‌های آماده

### فلسفه: حداکثر استفاده از کتابخانه‌های موجود، حداقل کد دستی

بر اساس تحقیقی که کردم، ترکیب زیر رو پیشنهاد می‌دم — هر کتابخانه یک مشکل خاص رو حل می‌کنه و همه رایگان هستند:

| کتابخانه                                 | نوع                  | کار                                                  | هزینه  |
| ---------------------------------------- | -------------------- | ---------------------------------------------------- | ------ |
| **FTS5 + `unicode61`**                   | Built-in SQLite      | ایندکس و BM25 ranking                                | رایگان |
| **`Intl.Segmenter`**                     | Built-in Node.js 20+ | تشخیص مرز کلمات برای فارسی/عربی/لاتین (مبتنی بر ICU) | رایگان |
| **`@persian-tools/persian-tools`**       | npm (MIT)            | نرمال‌سازی حروف فارسی (ک ← ك، ی ← ي، اعداد)          | رایگان |
| **`String.prototype.normalize('NFKC')`** | Built-in JS          | نرمال‌سازی یونیکد                                    | رایگان |

### چرا این ترکیب؟

1. **FTS5 + `unicode61`** — Built-in، بدون dependency. `remove_diacritics=2` برای اعراب فارسی/عربی. `tokenchars` برای "گیت‌هاب". `prefix='2 3'` برای جستجوی پیشوندی.

2. **`Intl.Segmenter`** — از ICU اپل/موزیلا استفاده می‌کنه. تنها راه استاندارد برای تشخیص مرز کلمات در یونیکد. جایگزین تابع دستی `tokenize()` که فقط whitespace می‌شناخت. کد دستی → صفر.

3. **`@persian-tools/persian-tools`** — کتابخانه محبوب و active برای پردازش متن فارسی. توابع `toPersianChars()`، `digitsEnToFa()`، `hasPersian()`. جایگزین کد دستی نرمال‌سازی.

4. **چرا ICU extension برای خود FTS5 نه؟** — نیاز به کامپایل C extension + توزیع `.dylib` مک/`.so` لینوکس/`.dll` ویندوز داره. `Intl.Segmenter` دقیقاً همون کار رو در سطح اپلیکیشن می‌کنه بدون درد build.

### معماری نهایی

```
Index Time:
  Article (raw)
    → normalize_fts() در SQLite (ثبت شده از طریق .function() API)
        ← @persian-tools/persian-tools (کتابخانه)
        ← String.normalize('NFKC') (built-in)
    → articles_fts_content table (INTEGER rowid)
    → FTS5 auto-sync (content= option)

Query Time:
  User Query
    → tokenizeQuery()
        ← Intl.Segmenter (کتابخانه - تشخیص مرز کلمات)
        ← @persian-tools/persian-tools (کتابخانه - نرمال‌سازی)
        ← String.normalize('NFKC') (built-in)
    → FTS5 MATCH query
    → JOIN articles
    → rank (BM25) + snippet()

Sync: SQLite triggers روی articles table → articles_fts_content → FTS5 auto-sync
```

### جزئیات پیاده‌سازی

#### 1. FTS5 Schema (`fts-schema.ts`)

```sql
-- Content table (برای FTS5 content= sync)
CREATE TABLE IF NOT EXISTS articles_fts_content (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  article_id TEXT NOT NULL UNIQUE,
  title TEXT NOT NULL,
  content TEXT NOT NULL
);

-- FTS5 virtual table با content=
CREATE VIRTUAL TABLE IF NOT EXISTS articles_fts USING fts5(
  title, content,
  content='articles_fts_content',
  content_rowid='id',
  tokenize='unicode61 remove_diacritics 2 tokenchars',
  prefix='2 3'
);

-- Triggers خودکار
CREATE TRIGGER IF NOT EXISTS trg_fts_insert
AFTER INSERT ON articles BEGIN
  INSERT INTO articles_fts_content(article_id, title, content)
  VALUES (NEW.id, normalize_fts(NEW.title), normalize_fts(NEW.content));
END;

CREATE TRIGGER IF NOT EXISTS trg_fts_delete
AFTER DELETE ON articles BEGIN
  DELETE FROM articles_fts_content WHERE article_id = OLD.id;
END;

CREATE TRIGGER IF NOT EXISTS trg_fts_update
AFTER UPDATE ON articles BEGIN
  UPDATE articles_fts_content
  SET title = normalize_fts(NEW.title), content = normalize_fts(NEW.content)
  WHERE article_id = NEW.id;
END;
```

#### 2. Text Normalizer (`text-normalizer.ts`)

```typescript
// فقط استفاده از کتابخانه‌ها — هیچ کد دستی
import { toPersianChars, digitsEnToFa } from "@persian-tools/persian-tools";

/** نرمال‌سازی متن برای ایندکس */
export function normalizeText(text: string): string {
	return toPersianChars(
		// کتابخانه: ک ← ك، ی ← ي
		digitsEnToFa(
			// کتابخانه: 123 ← ۱۲۳
			text.normalize("NFKC"), // built-in JS
		),
	);
}

/** جداسازی کلمات با Intl.Segmenter (ICU-based) */
export function tokenizeQuery(query: string): string[] {
	const normalized = normalizeText(query).toLowerCase();
	const words: string[] = [];
	const segmenter = new Intl.Segmenter(["en", "fa", "ar"], {
		granularity: "word",
	});
	for (const seg of segmenter.segment(normalized)) {
		if (seg.isWordLike && seg.segment.length >= 2) {
			words.push(seg.segment);
		}
	}
	return [...new Set(words)].slice(0, 12);
}
```

**حذف کد دستی:** تابع قدیمی `tokenize()` که ۵۰ خط کد دستی با stop-word و regex داشت، با ۲۰ خط کد که فقط کتابخانه صدا می‌زنه جایگزین می‌شه.

#### 4. Search Service - جستجوی FTS5

```typescript
async keyword(q: string, opts = {}): Promise<SearchResult> {
  const maxResults = Math.min(100, Math.max(1, opts.limit ?? 20));
  const terms = tokenizeQuery(q);
  if (terms.length === 0) return { query: q, hits: [], totalMatches: 0 };

  // FTS5 query: implicit AND, fallback به OR
  const andQuery = terms.map(t => `"${escapeFts(t)}"`).join(' ');
  const orQuery = terms.map(t => `"${escapeFts(t)}"`).join(' OR ');

  // امتحان AND اول
  const hits = this.runFtsQuery(andQuery, maxResults, opts.periodMs);
  if (hits.length > 0) return hits;

  // Fallback به OR
  const orHits = this.runFtsQuery(orQuery, maxResults, opts.periodMs);
  return orHits;
}

private runFtsQuery(ftsQuery: string, limit: number, periodMs?: number): SearchResult {
  const since = periodMs ? Date.now() - periodMs : null;
  const rows = this.db.rawDb.prepare(`
    SELECT a.*, rank, snippet(articles_fts, 1, '…', '…', '…', 64) AS highlight
    FROM articles_fts fts
    JOIN articles_fts_content c ON c.id = fts.rowid
    JOIN articles a ON a.id = c.article_id
    WHERE articles_fts MATCH ?
      AND (? IS NULL OR a.collected_at >= ?)
    ORDER BY rank
    LIMIT ?
  `).all(ftsQuery, since, since, limit);
  // ... به SearchHit تبدیل کن
}
```

### Edge Cases

| مورد                         | رفتار                                             |
| ---------------------------- | ------------------------------------------------- |
| Query خالی                   | return empty (مثل الان)                           |
| FTS5 chars خاص (`^ * " ( )`) | Escaping با escapeFts()                           |
| AND صفر نتیجه                | Fallback به OR                                    |
| متن فقط فارسی                | Intl.Segmenter + unicode61 درست کار می‌کنه        |
| متن مخلوط "AI گیت‌هاب"       | هر دو به درستی tokenize و MATCH می‌شن             |
| **"github" → "گیت‌هاب"**     | **FTS5 حل نمی‌کنه** — Ask AI mode پوشش می‌ده      |
| Article خالی                 | content='' → FTS5 بی‌تاثیر، در ایندکس نیست        |
| حجم زیاد                     | FTS5 ایندکس شده (O(log n))                        |
| Migrate مجدد                 | CREATE IF NOT EXISTS + backfill فقط رکوردهای جدید |

### جمع‌بندی تغییرات کد

| کتابخانه              | LOC دستی قدیم                                | LOC جدید (با کتابخانه)                 |
| --------------------- | -------------------------------------------- | -------------------------------------- |
| tokenize()            | 50 خط دستی (lower, split, regex, stop-words) | 15 خط (Intl.Segmenter + persian-tools) |
| scoreHit()            | 20 خط دستی (heuristic TF)                    | 0 خط (BM25 rank از FTS5)               |
| makeHighlight()       | 20 خط دستی (slicing)                         | 0 خط (snippet() از FTS5)               |
| Persian normalization | وجود ندارد                                   | 0 خط (persian-tools)                   |

**کل کد دستی جدید:** حدود ۶۰ خط (تنظیمات FTS5 + فراخوانی کتابخانه‌ها)
**کد دستی حذف شده:** حدود ۱۰۰ خط (tokenize, scoreHit, makeHighlight, ...)
**کتابخانه‌های اضافه شده:** ۱ عدد (`@persian-tools/persian-tools`)
