## Full-Text Search with Hybrid Approach + Ready-made Libraries

### Philosophy: Maximal Use of Existing Libraries, Minimal Manual Code

Based on my research, I recommend the following combination—each library solves a specific problem, and all are free:

| Library                                  | Type                 | Purpose                                                      | Cost |
| ---------------------------------------- | -------------------- | ------------------------------------------------------------ | ---- |
| **FTS5 + `unicode61`**                   | Built-in SQLite      | Indexing and BM25 ranking                                    | Free |
| **`Intl.Segmenter`**                     | Built-in Node.js 20+ | Detects word boundaries for Persian/Arabic/Latin (ICU-based) | Free |
| **`@persian-tools/persian-tools`**       | npm (MIT)            | Normalizes Persian letters (ک ← ك , ی ← ي, digits)           | Free |
| **`String.prototype.normalize('NFKC')`** | Built-in JS          | Unicode normalization                                        | Free |

### Why this combination?

1. **FTS5 + `unicode61`** — Built-in, no dependency. `remove_diacritics=2` for Persian/Arabic diacritics. `tokenchars` for "GitHub". `prefix='2 3'` for prefix search.

2. **`Intl.Segmenter`** — Uses ICU from Apple/Mozilla. The only standard way to detect word boundaries in Unicode. Replaces the manual `tokenize()` function that only recognized whitespace. Manual code → zero.

3. **`@persian-tools/persian-tools`** — A popular and active library for Persian text processing. Functions: `toPersianChars()`, `digitsEnToFa()`, `hasPersian()`. Replaces manual normalization code.

4. **Why not the ICU extension for FTS5 itself?** — Requires compiling a C extension + distributing `.dylib` for Mac/`.so` for Linux/`.dll` for Windows. `Intl.Segmenter` does the exact same at the application layer without build headaches.

### Final Architecture

```
Index Time:
  Article (raw)
    → normalize_fts() in SQLite (registered via .function() API)
        ← @persian-tools/persian-tools (library)
        ← String.normalize('NFKC') (built-in)
    → articles_fts_content table (INTEGER rowid)
    → FTS5 auto-sync (content= option)

Query Time:
  User Query
    → tokenizeQuery()
        ← Intl.Segmenter (library - word boundary detection)
        ← @persian-tools/persian-tools (library - normalization)
        ← String.normalize('NFKC') (built-in)
    → FTS5 MATCH query
    → JOIN articles
    → rank (BM25) + snippet()

Sync: SQLite triggers on articles table → articles_fts_content → FTS5 auto-sync
```

### Implementation Details

#### 1. FTS5 Schema (`fts-schema.ts`)

```sql
-- Content table (for FTS5 content= sync)
CREATE TABLE IF NOT EXISTS articles_fts_content (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  article_id TEXT NOT NULL UNIQUE,
  title TEXT NOT NULL,
  content TEXT NOT NULL
);

-- FTS5 virtual table with content=
CREATE VIRTUAL TABLE IF NOT EXISTS articles_fts USING fts5(
  title, content,
  content='articles_fts_content',
  content_rowid='id',
  tokenize='unicode61 remove_diacritics 2 tokenchars',
  prefix='2 3'
);

-- Automatic Triggers
CREATE TRIGGER IF NOT EXISTS trg_fts_insert
AFTER INSERT ON articles BEGIN
  INSERT INTO articles_fts_content(article_id, title, content)
  VALUES (NEW.id, normalize_fts(NEW.title), normalize_fts(NEW.content));
END;

CREATE TRIGGER IF NOT EXISTS trg_fts_delete
AFTER DELETE ON articles BEGIN
  DELETE FROM articles_fts_content WHERE article_id = OLD.id;
END;

CREATE TRIGGER IF NOT EXISTS trg_fts_update
AFTER UPDATE ON articles BEGIN
  UPDATE articles_fts_content
  SET title = normalize_fts(NEW.title), content = normalize_fts(NEW.content)
  WHERE article_id = NEW.id;
END;
```

#### 2. Text Normalizer (`text-normalizer.ts`)

```typescript
// Only use libraries — no manual code
import { toPersianChars, digitsEnToFa } from "@persian-tools/persian-tools";

/** Normalize text for indexing */
export function normalizeText(text: string): string {
	return toPersianChars(
		// Library: ک ← ك, ی ← ي
		digitsEnToFa(
			// Library: 123 ← ۱۲۳
			text.normalize("NFKC"), // built-in JS
		),
	);
}

/** Split words with Intl.Segmenter (ICU-based) */
export function tokenizeQuery(query: string): string[] {
	const normalized = normalizeText(query).toLowerCase();
	const words: string[] = [];
	const segmenter = new Intl.Segmenter(["en", "fa", "ar"], {
		granularity: "word",
	});
	for (const seg of segmenter.segment(normalized)) {
		if (seg.isWordLike && seg.segment.length >= 2) {
			words.push(seg.segment);
		}
	}
	return [...new Set(words)].slice(0, 12);
}
```

**Manual code removal:** The old `tokenize()` function, with 50 lines of manual code for stop-words and regex, is replaced by 20 lines that only call libraries.

#### 4. Search Service - FTS5 Search

```typescript
async keyword(q: string, opts = {}): Promise<SearchResult> {
  const maxResults = Math.min(100, Math.max(1, opts.limit ?? 20));
  const terms = tokenizeQuery(q);
  if (terms.length === 0) return { query: q, hits: [], totalMatches: 0 };

  // FTS5 query: implicit AND, fallback to OR
  const andQuery = terms.map(t => `"${escapeFts(t)}"`).join(' ');
  const orQuery = terms.map(t => `"${escapeFts(t)}"`).join(' OR ');

  // Try AND first
  const hits = this.runFtsQuery(andQuery, maxResults, opts.periodMs);
  if (hits.length > 0) return hits;

  // Fallback to OR
  const orHits = this.runFtsQuery(orQuery, maxResults, opts.periodMs);
  return orHits;
}

private runFtsQuery(ftsQuery: string, limit: number, periodMs?: number): SearchResult {
  const since = periodMs ? Date.now() - periodMs : null;
  const rows = this.db.rawDb.prepare(`
    SELECT a.*, rank, snippet(articles_fts, 1, '…', '…', '…', 64) AS highlight
    FROM articles_fts fts
    JOIN articles_fts_content c ON c.id = fts.rowid
    JOIN articles a ON a.id = c.article_id
    WHERE articles_fts MATCH ?
      AND (? IS NULL OR a.collected_at >= ?)
    ORDER BY rank
    LIMIT ?
  `).all(ftsQuery, since, since, limit);
  // ... convert to SearchHit
}
```

### Edge Cases

| Case                             | Behavior                                         |
| -------------------------------- | ------------------------------------------------ |
| Empty Query                      | return empty (as before)                         |
| Special FTS5 chars (`^ * " ( )`) | Escaping via escapeFts()                         |
| AND returns zero results         | Fallback to OR                                   |
| Only Persian text                | Intl.Segmenter + unicode61 work correctly        |
| Mixed text "AI گیت‌هاب"          | Both are correctly tokenized and MATCHed         |
| **"github" → "گیت‌هاب"**         | **FTS5 can't handle this** — Ask AI mode covers  |
| Empty Article                    | content='' → no effect on FTS5, not indexed      |
| Large volume                     | Indexed FTS5 (O(log n))                          |
| Remigrate                        | CREATE IF NOT EXISTS + backfill only new records |
